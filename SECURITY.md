# Security Policy (Individual)

## Reporting a Security Issue
If you believe you have found a security vulnerability, please report it privately to:

- **Email:** timmytheonlinegirl@hotmail.com

## What to Include in Your Report
To help me evaluate the issue quickly, please include:

- A clear description of the vulnerability
- Steps to reproduce (if applicable)
- Affected version(s) (if known)
- Expected vs. actual behavior
- Any relevant logs, screenshots, or proof-of-concept

## Response Process
After receiving your report, I will:

1. Acknowledge receipt as soon as possible.
2. Assess severity and impact.
3. Confirm whether the issue is valid and whether it affects this repository.
4. Work on a fix or mitigation.

## Responsible Disclosure & Timeline
- I aim to provide an initial response within **72 hours** (when feasible).
- If a fix is possible, I will work toward a release or mitigation as soon as I reasonably can.
- If I cannot reproduce the issue, I will communicate what I found and what additional information (if any) would be needed.

## Public Disclosure
I will coordinate any public disclosure with you (when appropriate). If I don’t hear back, I may publish details once a fix/mitigation is available.
